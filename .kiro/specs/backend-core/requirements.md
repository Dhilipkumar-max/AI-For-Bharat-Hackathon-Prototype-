## Introduction

Backend Core provides the server-side infrastructure for the AI Vidya for Bharat platform.

The backend exposes REST APIs that allow the frontend to:

- generate AI-powered courses
- retrieve generated courses
- list previously generated courses
- interact with the AI tutor (AI Guru)

The backend integrates with AWS Bedrock to generate structured course content and tutor responses.

The system is designed as a lightweight backend optimized for hackathon development and demo stability.

There is no authentication system. All users operate in guest mode.

Courses are stored in a lightweight persistent database so generated courses remain available even after server restart.

---

# System Overview

The Backend Core consists of the following subsystems:

- API_Service
- Course_Service
- Chat_Service
- AI_Service
- Database_Layer
- Config_Manager
- Error_Handler

These subsystems together provide course generation, course retrieval, and AI tutoring capabilities.

---

# Glossary

Backend_Core  
The server-side application that provides API services for the platform.

API_Service  
The HTTP server that exposes REST endpoints for frontend interaction.

Course_Service  
The subsystem responsible for generating and retrieving courses.

Chat_Service  
The subsystem responsible for AI tutor conversations.

AI_Service  
The subsystem responsible for communication with AI providers such as AWS Bedrock.

Database_Layer  
The persistence layer that stores generated courses.

Client  
The frontend application making API requests.

Course  
A structured learning resource generated by AI.

Chapter  
A section within a course containing learning content.

Message  
A user question sent to the AI tutor.

AI_Provider  
External service that generates AI responses (AWS Bedrock).

Request_ID  
A unique identifier used to trace API requests.

# Core Functional Requirements

## 1. Course Generation

The system must allow the client to generate a structured learning course using AI.

Endpoint:

POST /api/generate-course

Request:

{
  topic: string
  language: string
}

The backend must:

1. validate the request
2. construct a structured AI prompt
3. send the prompt to AWS Bedrock
4. receive structured course data
5. store the course in the database
6. return the course identifier

Response:

{
  courseId: string
}

---

## 2. Retrieve Course

The system must allow the client to retrieve a previously generated course.

Endpoint:

GET /api/course/:id

Response:

{
  id
  title
  topic
  language
  overview
  learning_outcomes
  chapters
  created_at
}

The backend must return the stored course without regenerating it.

---

## 3. List Generated Courses

The system must provide a list of all previously generated courses.

Endpoint:

GET /api/courses

Response:

[
  {
    id
    title
    topic
    language
    created_at
  }
]

This endpoint is used by the Course Dashboard (AI Learning Library).

---

## 4. Delete Courses

The system must allow clearing all stored courses.

Endpoint:

DELETE /api/courses

This is used by the Settings page to reset the demo environment.

---

## 5. AI Tutor (AI Guru)

The system must allow users to ask questions related to a generated course.

Endpoint:

POST /api/chat

Request:

{
  message: string
  courseId: string
}

The backend must:

1. retrieve the course context
2. construct an AI tutor prompt
3. send the prompt to AWS Bedrock
4. return the generated response

Response:

{
  reply: string
}

---

# AI Integration Requirements

The backend must integrate with AWS Bedrock for AI generation.

The AI_Service must support:

- course generation
- AI tutor responses

The AI prompt must request structured JSON output.

Example prompt:

Generate a beginner-friendly course about "{topic}".

Language: {language}

Return JSON format:

{
  overview: "",
  learning_outcomes: [],
  chapters: [
    {
      title: "",
      content: ""
    }
  ]
}

---

# Database Requirements

The backend must store generated courses in a persistent database.

SQLite is recommended for simplicity.

Table: courses

Fields:

id
title
topic
language
overview
learning_outcomes (JSON)
chapters (JSON)
created_at

The database must persist data across server restarts.

---

# Error Handling Requirements

The backend must return structured error responses.

Example:

{
  error: "Course generation failed"
}

Errors must be handled for:

- invalid input
- AI provider failure
- database failure

---

# Configuration Requirements

Environment variables must be used for configuration.

Required variables:

AWS_REGION
AWS_BEDROCK_MODEL_ID
DATABASE_PATH

The Config_Manager must load and validate environment variables at server startup.

---

# Performance Requirements

The backend must:

- handle asynchronous AI requests
- avoid blocking operations
- respond quickly to API requests

Course generation may take several seconds due to AI processing.

---

# Security Requirements

Basic input validation must be implemented.

Topic input must:

- be at least 3 characters
- not exceed reasonable length

Language must be validated against allowed values.

---

# Non Functional Requirements

The backend must:

- run locally using Node.js
- start with `npm run dev`
- integrate easily with the existing frontend
- remain lightweight and easy to maintain
